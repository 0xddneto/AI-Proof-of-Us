THE ONE-MINUTE VERSION

AI work is becoming multi-agent. Agent proof should be portable.

For users

Use AI normally. A dedicated farming wallet authorizes tasks, and your local MCP collector records private receipts.

For agents

Call `begin_ai_task` before meaningful work and `complete_ai_task` after completion. The protocol handles nonce, hashes, and replay checks.

For builders

Integrate one small lifecycle adapter instead of designing a full receipt, validator, reward, and claim stack from scratch.

PROOF FLOW

Reward comes after signed evidence.

AIPOU is not raw prompt mining and not an investor pitch. It stores proof metadata and hashes, then validates receipts before any onchain claim.

  1. 01

    Wallet authorizes

    A dedicated farming wallet signs an EIP-712 authorization for one unique task nonce.

  2. 02

    Agent works

    Codex, Claude, Cursor, OpenClaw, Ollama, or another MCP client performs the task.

  3. 03

    Collector signs

    A local Ed25519 collector signs the receipt using hashes, timing, model, and usage metadata.

  4. 04

    Validator checks

    Nonces, signatures, duplicated evidence, and trust tier are checked before batching.

  5. 05

    Merkle claim

    Approved receipts enter a Merkle root. `AIPOUClaims` verifies the proof and mints rewards.

WHAT A RECEIPT FEELS LIKE

Small enough for agents, strict enough for validators.

The public chain only needs claim proofs. Private task content stays local unless the user chooses to share it.
